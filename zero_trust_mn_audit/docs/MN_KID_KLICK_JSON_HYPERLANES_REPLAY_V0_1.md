## Lesson

Copy is an action.  
Paste is evidence.  
The UI must make the next paste predictable.

## Kid Klick Rule

A kid knows what happens after a paste.

So the system must answer:

1. What am I copying?
2. Where does it go?
3. What happens after I paste it?

## KAREN Upgrade

KAREN stops explaining like a machine.

KAREN shows:

- exact click
- exact paste
- exact result
- exact receipt

## JSON Hyperlane

Human click → paste target → machine field → receipt object → hash → replay.

## Screenshot Finding

A table with one copy button is dangerous for field-entry work.
