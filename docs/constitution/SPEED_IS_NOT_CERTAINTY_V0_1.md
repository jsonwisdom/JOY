## Purpose

Define the boundary between accelerated hypothesis generation and validated knowledge.

Models can compress the time from question to candidate answer. They cannot replace empirical or logical verification.

## Core Rule

```text
Mistaking speed for certainty is the trap.
Using speed to test more things, more rigorously, is the leverage.
```

## Model Role

A model is an accelerator of hypothesis generation.

It may help produce:

- candidate answers
- possible paths
- research lanes
- source trails
- worker tasks
- draft schemas
- replay instructions

## Validation Role

Validation still requires accountability through:

- replay
- replication
- falsification
- source inspection
- deterministic checks
- preserved evidence chains
- independent review

## Operating Doctrine

```text
Fast output is not proof.
A candidate answer is not verification.
A plausible route is not a receipt.
A model summary is not replay.
```

Use speed to widen the test surface.
Do not use speed to skip the test.

## Worker Rule

Workers may move fast when they are producing leads, drafts, logs, and paths.

Workers must slow down when a state would be promoted.

```text
Generate fast.
Promote slow.
Replay always.
```
