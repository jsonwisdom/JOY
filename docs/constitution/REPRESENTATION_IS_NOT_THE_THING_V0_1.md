## Purpose

Define the anti-substitution rule for model outputs, drafts, source trails, commits, and verification claims.

The representation or record of a thing is not the thing itself, especially when trust or truth is at stake.

## Core Rule

```text
The map is not the territory.
The record is not the event.
The model answer is not verification.
The commit is not external truth.
Replay is the test.
```

## Boundary Lines

```text
A candidate is not a conclusion.
A draft is not a receipt.
A source trail is not preservation.
A commit is not external truth.
A model answer is not verification.
```

## Why It Matters

Models operate in the realm of plausible representation.

Verification lives outside that realm, in replay, preservation, independent anchoring, source inspection, and reproducible checks.

## Substitution Errors

Do not substitute:

- candidate for conclusion
- draft for receipt
- citation for preservation
- commit for correctness
- model fluency for evidence
- public signal for proof
- timestamp for legal conclusion
- visibility for authority

## Valid Promotion Path

```text

  ↓
```
