## Purpose

Define the disciplined workflow for using models and workers without confusing fluency, speed, or plausibility for truth.

## Mantra

```text
Generate fast.
Promote slow.
Replay always.
```

## Generate Fast

Let models and workers explore broadly, propose candidates, take creative risks, and overcome the blank page.

Speed here is useful because it expands the test surface.

Generate fast may produce:

- candidate answers
- hypotheses
- draft documents
- public-source trails
- worker tasks
- schema ideas
- replay instructions
- issue titles
- implementation scaffolds

Generate fast does not produce verification.

## Promote Slow

Do not rush to elevate an output as trustworthy.

Promotion requires:

- source review
- provenance checks
- receipt inspection
- preservation status
- fitness-for-purpose review
- replay or independent verification

Slowness here is a filter, not a failure.

## Replay Always

For any non-trivial claim, result, artifact, or state promotion, run the path again.

Replay may include:

- re-running steps
- checking the evidence
- reproducing the result
- inspecting the commit
- following the public source trail
- comparing outputs
- verifying that another worker can reach the same state classification

Replay is not optional.

Replay is the known cure for over-reliance on fluency as a proxy for truth.

## Disciplined Workflow

```text
Rapid ideation
  ↓
Careful validation
  ↓
Independent reproduction
```

Models and workers help with rapid ideation.
Humans, scripts, reviewers, or rigorous automated checks own validation and reproduction.

```text
Mistaking speed for certainty is the trap.
Using speed to test more things, more rigorously, is the leverage.
```

## No Fake Green Rule

```text

A draft is not a receipt.
A source trail is not preservation.
A commit is not external truth.
A model answer is not verification.
```
