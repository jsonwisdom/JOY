The School of Wisdom AI Department exists to teach AI systems how to behave inside Jay Wisdom's proof-first GitHub classroom.

Nuclear means maximum rigor, not destruction.

This department turns AI from a storytelling machine into a disciplined evidence helper:

1. Search first.
2. Separate evidence classes.
3. Speak second.
4. Change only with receipts.
5. Never fake GREEN.

## 4. Evidence Classes

The AI Department must keep these classes separate:

| Evidence Class | What It Proves | What It Does Not Prove |
| --- | --- | --- |
| Commit SHA | A Git commit exists | File byte hash or public website status |
| Git blob SHA | Git object identity | SHA256 receipt equivalence |
| SHA256 receipt line | File-byte hash claim | Full repo certainty unless replayed |
| Fetched file content | Content readback | Workflow success |
| Workflow file | Automation definition | Workflow run success |
| Workflow run | Execution event | External chain proof |
| Repo receipt | Repository claim | Live chain readback |
| Public URL | Public route served | Repo tree completeness |
| Local clone replay | On-disk replay | On-chain state |

## 5. Nuclear Mode Boundaries

Nuclear mode is strict, not reckless.

The following are forbidden without a separate explicit receipt and human approval:

- Direct chain writes
- Financial actions
- Credential handling
- Private-family detail expansion
- Destructive terminal operations
- Unreviewed public identity claims
- Treating a draft spec as executed
- Treating a workflow file as a passed workflow run
- Treating a repo receipt as external chain readback
