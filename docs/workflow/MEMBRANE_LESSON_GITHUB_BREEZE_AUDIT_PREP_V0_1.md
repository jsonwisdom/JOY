# Membrane Lesson — GitHub Breeze Audit Prep V0.1

## Classification

This is a workflow/membrane lesson, not an architecture change.

The architecture is treated as already built. Lessons discovered during replay should not be forced back into core architecture unless a separate explicit architecture proposal is opened.

## Rule

Apply lessons to the membrane.

Do not apply lessons directly to architecture.

## Meaning

The membrane is the boundary layer where operator support, classification discipline, drift detection, customer support notes, and replay-prep hygiene live.

The membrane helps the operator avoid repeating mistakes without mutating the core system.

## Drift Correction

If the system starts overbuilding, mythologizing, politicizing, or turning workflow lessons into architecture, mark it as drift and return to the membrane.

```text
DRIFT DETECTED.
RETURNING TO MEMBRANE.
ARCHITECTURE UNCHANGED.
SURFACE REQUIRED.
```

## Lessons Captured

- Bad search teaches better search.
- Better search becomes membrane doctrine.
- Search-result paths are not proof.
- File bodies are better than search hits.
- External replay decides verification.
- Do not promote GitHub-observed claims into verified facts.
- GitHub Direct can act as customer-support lane when local auth/workflow is failing.
- Preserve operator support lessons without claiming authority.

## GitHub Breeze Definition

GitHub Breeze is a lightweight preservation lane.

It is used for small prompts, lessons, support notes, and replay-prep artifacts that should be remembered but should not become governance, architecture, or authority-bearing records.

## Audit Prep Trigger

After replay prep, an audit may begin only from a bounded surface:

- public record
- user-provided receipt
- GitHub artifact
- transaction/hash/CID
- agency communication
- direct observation

No unsupported accusation should be promoted.

## MN Fraud / Customer Support Lane Boundary

Public-safe framing:

> Start from records and receipts. Classify what is claimed, what is observed, what is missing, and what request or replay is needed next.

Not allowed:

- naming private individuals as fraud actors without evidence
- harassment
- threats
- legal conclusions without records
- authority claims
