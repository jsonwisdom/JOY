## Purpose

This plan defines a root-level GitHub Direct workflow for `jaywisdom.base.eth` that treats time as a loop, records each loop as a receipt, and sends only opt-in timed reports.

The style is simple: hatch clock, reset button, visible receipt.

## Core Rule

Do not touch the Matrix until the loop produces receipts.

Receipts before claims. Hashes before stories. Public commits before promotion.

## Loop Model

Each loop has five steps:

1. Scan public repo metadata.
2. Generate JSON report.
3. Compare against previous report.
4. Mark PASS, WARN, or HOLD.
5. Send opt-in summary email only if a recipient explicitly approved it.

## Time Loops

- 6-hour Sentinel Loop: confirms builder boundary workflows are still present.
- Daily Calm Report Loop: summarizes repo changes and pending external receipts.
- Weekly Reputation Ledger Loop: summarizes public commits, schemas, and open gates.
- Manual Hatch Button: operator-triggered report before any major Matrix move.

## Email Boundary

Emails are summaries, not pressure systems.

Rules:

- No automatic emails to unapproved recipients.
- No repeated escalation loops.
- No age-targeted language.
- No private claims without receipts.
- Every email must cite the GitHub report path it summarizes.
