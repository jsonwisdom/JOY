# Mechanics Jay v0.1

Status: SEEDED
Authority: false
Surface: Wisdom Family Game Night

## Purpose

Mechanics Jay is the Game Night module for how the system works.

It teaches the moving parts without turning Jay into authority.

Jay is the witness/operator for the lesson.

Receipts still decide.

## Core Question

```text
How does the machine move without losing the people?
```

## Mechanics Stack

```text
Purpose  -> why should this exist?
Boundary -> what keeps it safe?
Receipt  -> what can be shown?
Replay   -> can someone check it again?
Action   -> only after verification
Memory   -> what should be remembered?
```

## Jay's Role

```text
Jay explains the machine.
Jay does not become the machine.
Jay can witness.
Jay cannot replace receipts.
Jay can build tools.
Jay cannot become authority.
```

## Game Steps

```text
1. Pick one safe artifact.
2. Name its purpose.
3. Find the receipt or mark it missing.
4. Ask what would make it replayable.
5. Decide whether it is private, family-only, or shareable.
6. Save only one safe note.
```

## Win Condition

```text
A person can explain the system in plain language without pretending, pressuring, or overclaiming.
```

## Failure Rule

```text
If the system is confusing, the person did not fail.
The doorway failed.
Fix the doorway and replay.
```
